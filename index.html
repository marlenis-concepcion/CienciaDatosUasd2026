Caso practico 2: Sakila CRUD/ORM nativo

Proyecto en Python para gestionar paises, ciudades, peliculas e inventario de MySQL Sakila usando POO, estructuras de datos, DbContext, repositorios, controladores, metricas descriptivas e importacion/exportacion CSV y JSON.

README del caso Informe PDF Checklist
Arquitectura

CRUD/ORM basado en POO

La solucion separa conexion, contexto, entidades, repositorios, controladores, servicios, estructuras de datos y menu de consola para mantener un flujo claro y ampliable.

  • DbContext centraliza repositorios, cache e historial.
  • Modelos: country, city, film e inventory.
  • ModelCollection representa listas de entidades.
  • Repositorios ejecutan create, read, update y delete.
  • Controladores organizan el flujo tipo MVC.
Quickstart

Un setup por sistema

Cada integrante puede levantar MySQL/Sakila con Docker, probar la conexion y abrir el menu interactivo sin configurar manualmente todo el entorno. 

cd caso_practico_2_sakila_crud_orm
./MAC_SH_QUICKSTART.sh
./LINUX_SH_QUICKSTART.sh
.\WINDOWS_QUICKSTART.ps1

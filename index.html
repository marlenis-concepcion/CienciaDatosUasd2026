Caso practico 2

Sakila CRUD/ORM nativo

Implementa un ORM propio en Python para gestionar paises, ciudades, peliculas e inventario sobre MySQL Sakila. El flujo usa DbContext, entidades, repositorios, controladores, servicios, cache e historial de operaciones.

Componentes clave

  • CRUD completo basado en POO.
  • Entidades: country, city, film e inventory.
  • ModelCollection como lista de entidades.
  • Importacion/exportacion CSV y JSON.
  • Media, rango, desviacion, varianza y covarianza.
  • 10 consultas SQL y unique constraints.

Quickstart

Un setup por sistema

El proyecto incluye scripts separados para que cada integrante pueda levantar MySQL/Sakila en Docker, verificar la conexion y abrir el menu CRUD/ORM sin configurar manualmente las variables de entorno.

Archivos disponibles

  • Mac: `MAC_SH_QUICKSTART.sh`.
  • Linux: `LINUX_SH_QUICKSTART.sh`.
  • Windows PowerShell: `WINDOWS_QUICKSTART.ps1`.
  • Base compartida: `setup_run_sakila_docker.sh`.

# Uso de agentes IA en el proyecto

## Objetivo

Usar agentes especializados para apoyar el trabajo colaborativo de Mccarthy Team en analisis tecnico, programacion, validacion, documentacion e informe APA/UASD.

## Principios

- La IA apoya, pero el equipo valida.
- Las decisiones tecnicas deben quedar documentadas.
- No se deben publicar credenciales, contrasenas ni datos privados.
- El codigo generado debe probarse antes de anexarlo.
- Los informes deben incluir citas verificables, no referencias inventadas.

## Flujo recomendado

1. El agente tecnico propone o revisa codigo.
2. El agente APA revisa estructura, citas, referencias y anexos.
3. El agente QA revisa ejecucion, pruebas, riesgos y evidencias.
4. El equipo humano aprueba la version final.

## Evidencia sugerida

En anexos puede incluirse una tabla breve:

| Agente | Funcion | Resultado revisado |
| --- | --- | --- |
| sakila_backend_agent | CRUD/ORM nativo | Arquitectura, repositorios, menu |

| qa_reviewer_agent | Calidad | Pruebas, riesgos, checklist |

# Checklist de entrega - Unidad 2

## Informe

- [ ] Tiene presentacion.
- [ ] Tiene resumen.
- [ ] Tiene abstract.
- [ ] Tiene tabla de contenido.
- [ ] El desarrollo tiene entre 5 y 10 paginas, si aplica.
- [ ] Incluye cita narrativa.
- [ ] Incluye cita parentetica.
- [ ] Incluye cita textual directa breve.
- [ ] Todas las citas tienen referencia.
- [ ] Todas las referencias estan citadas.
- [ ] Las referencias estan en APA 7.

## Caso practico 2

- [ ] El CRUD/ORM nativo conecta a MySQL Sakila.
- [ ] Permite crear registros.
- [ ] Permite consultar por ID.
- [ ] Permite listar registros.
- [ ] Permite actualizar registros.
- [ ] Permite eliminar registros de prueba.
- [ ] Evidencia POO y estructuras de datos.
- [ ] Hay capturas o logs de ejecucion.

## Caso practico 3

- [ ] Los siete CSV de OULAD estan en `data/raw/`.
- [ ] El esquema SQL tiene PK.
- [ ] El esquema SQL tiene FK.
- [ ] El esquema SQL tiene restricciones de dominio.
- [ ] El ETL genera validaciones de calidad.
- [ ] Hay evidencias de limpieza.
- [ ] Hay EDA extendido con tablas y graficas.
- [ ] Hay consultas de validacion.

## Calidad y seguridad

- [ ] No hay contrasenas en el codigo.
- [ ] Las credenciales se pasan por variables de entorno.
- [ ] El proyecto tiene README general.
- [ ] Cada caso tiene README propio.
- [ ] Las carpetas vacias tienen README o `.gitkeep`.
- [ ] Se declara el uso de agentes IA.
- [ ] Se reviso ortografia y consistencia.
